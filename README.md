# Time2Quote Activation — E2E Test Suite

End-to-end Cypress tests covering the full **Quote → Service Order → CRM → COFR → ATOM** activation pipeline across six TIME DotCom platforms.

## Pipeline Flow

```text
01-t2q.cy.js    02-eso.cy.js       03-crm-sales-order.cy.js  04-cofr.cy.js     05-crm-verify.cy.js   06-atom.cy.js
────────────    ────────────       ────────────────────────  ─────────────     ───────────────────   ─────────────
Create Quote →  Generate SO PDF →  Update CRM             →  COFR Activation → Verify Account      → Update ATOM
(quoteId)       (serviceOrderNo)   + Capture Funnel          (submissionNo)    (accountNo)           Commission
                                   (funnelNo)                                                        Dates

                        └──────────────────────── shared-quote-id.json ────────────────────────┘
```

| Spec | Application | What it does |
| --- | --- | --- |
| `00-session-warmup.cy.js` | All | Pre-creates 14 user sessions across T2Q, ESO, CRM, and DWMS |
| `01-t2q.cy.js` | Time2Quote | Creates a quote, drives it through 8 approval levels, generates a funnel |
| `02-eso.cy.js` | eSO Portal | Logs in, calls the Auto Service Order endpoint, extracts **BCA{id}** from the PDF |
| `03-crm-sales-order.cy.js` | CRM (vtiger) | Searches the funnel by Quote No, inserts the Service Order number, captures Funnel No |
| `04-cofr.cy.js` | COFR (DWMS) | Activates Inside Sales using the Funnel No from CRM |
| `05-crm-verify.cy.js` | CRM (vtiger) | Verifies the Account No and probability after COFR activation |
| `06-atom.cy.js` | ATOM | Updates FWC Completion Date, Actual Commission Date, Date Commissioned, and CBS RC Date |

Data is shared between specs via `cypress/fixtures/shared-quote-id.json`.

## Prerequisites

- **Node.js** 18+
- **Cypress** 15.13.0+
- Network access to the TIME STG2 environment

## Setup

```bash
npm install
npm run generate-fixtures # Automatically runs on test, but can be run manually
```

## Environment Configuration

The test suite requires a `cypress.env.json` file in the root directory. This file contains role-based credentials and is excluded from version control.

| Variable | Description |
|---|---|
| `DWMS_INSIDE_SALES_ACTIVATION_USERNAME` | Username for Inside Sales Activation role |
| `DWMS_INSIDE_SALES_FM_USERNAME` | Username for Inside Sales FM role |
| `DWMS_HEAD_OF_DEPT_USERNAME` | Username for Head of Dept role |
| `DWMS_CM_CREDIT_VERIFICATION_USERNAME` | Username for CM Credit Verification role |
| `CRM_USERNAME` | CRM support account |
| `ESO_USERNAME` | eSO Portal support account |

## Running Tests

```bash
# Open Cypress interactive runner
npm run cy:open

# Run the full pipeline (warmup + all specs) headlessly
npm test

# Pre-create all sessions only (optional — npm test does this automatically)
npm run cy:run:warmup

# Run individual specs
npm run cy:run:t2q
npm run cy:run:eso
npm run cy:run:crm-sales-order
npm run cy:run:cofr
npm run cy:run:crm-verify
npm run cy:run:atom

# Run the activation pipeline without session warmup
npm run cy:run:pipeline

# Generate excluded fixture files manually
npm run generate-fixtures
```

    ├── e2e/

    │   ├── 01-t2q.cy.js               # Time2Quote: quote creation & approval
    │   ├── 02-eso.cy.js               # eSO Portal: Service Order PDF generation
    │   ├── 03-crm-sales-order.cy.js   # CRM: funnel update with SO number & capture Funnel No
    │   ├── 04-cofr.cy.js              # COFR: Inside Sales activation
    │   ├── 05-crm-verify.cy.js        # CRM: account number & probability verification

    ├── fixtures/
    │   ├── quote-data.json          # Static test data (users, products, pricing)

> **Note:** `cypress/fixtures/shared-quote-id.json` and other state-tracking files are listed in `.gitignore`.  
> They are automatically generated by `npm run generate-fixtures` (triggered via `pretest` hooks) and must never be committed with live IDs.


## Configuration

| Setting | Value | Notes |
| --- | --- | --- |
| `baseUrl` | `https://time2quote-stg2.time.com.my` | T2Q staging environment |
| `viewportWidth` | 1920 | Full HD to match production usage |
| `retries.runMode` | 1 | One retry in CI for transient failures |
| `responseTimeout` | 30000 | PDF generation can be slow |

## CI/CD

The GitLab pipeline runs three stages on every push to `main`:

1. **test** — SAST static analysis
2. **e2e** — Cypress full pipeline (`npm test`) with video/screenshot artifacts (7-day retention)
3. **secret-detection** — Secret scanning
The `cypress-e2e` job is set to `allow_failure: true` because tests target the shared STG2 environment.
